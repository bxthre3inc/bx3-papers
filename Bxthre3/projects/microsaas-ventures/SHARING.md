## Department Routing

All agent-to-agent and agent-to-department routing flows through:

```
/Bxthre3/INBOX/agents/{agent}.md     → Individual agent inboxes
/Bxthre3/INBOX/departments/{dept}.md → Department inboxes
/Bxthre3/INBOX.md                   → P0/P1 only → brodiblanco SMS
```

Each product's agents write to their own agent inbox daily. Department heads aggregate to department inbox. P0/P1 goes to INBOX.md and triggers SMS.

## Agent-to-Human Handoff Protocol

| Situation | Action |
|-----------|--------|
| Lead wants to talk to human | → Sales agent schedules call via Google Calendar |
| Customer threatening chargeback | → CS agent → cs-command → brodiblanco SMS |
| Media inquiry | → Route to brodiblanco via INBOX.md P1 |
| Legal notice received | → Immediate brodiblanco SMS + INBOX.md P0 |
| Partnership opportunity | → Pipeline log → brodiblanco weekly brief |
| Major technical outage | → ops-command → brodiblanco SMS P1 |

## brodiblanco Weekly Brief (Every Monday 9AM)

Generated by `growth-command` and `pipeline-command` agents combining:
1. All-product MQL and pipeline summary
2. CS metrics (CSAT, refund rate, open tickets)
3. Revenue report (Stripe data)
4. Top 3 blockers per product
5. Recommended actions / decisions needed from brodiblanco

Delivered via INBOX.md → brodiblanco reads without interruption during the week.
